Heuristic Evaluation

This test is done in relation to Nielsen's 10 usability heuristics:

Heuristic ID Heuristic description Result
1 Visibility of system status - the design keeps users informed and provides appropriate feedback SUCCESS
2 Match between the system and real world - use words, phrases and concepts familiar to the user SUCCESS
3 User Control and Freedom - show a way back if the user accidentally presses a button SUCCESS
4 Consistency and Standards - users are not confused of words, situations or actions meaning the same thing SUCCESS
5 Error prevention - eliminate either by design or by implimentation any errors that may occur in the app SUCCESS
6 Recognition rather than call - minimize the user's memory load SUCCESS
7 Flexibility and efficiency of use - provide shortcuts to provide through which users can speed up their navigation SUCCESS
8 Aesthetic and minimalistic design - no useless or extra information SUCCESS
9 Help users recognise, diagnose and recover from errors - show error messages SUCCESS
10 Help and documentation - provide the user with help or some documentation to guide to use your app FAIL
